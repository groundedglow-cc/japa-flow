# JapaFlow 0.1.6 产品功能说明书（练习组提交与错题集复练版）

## 一、版本定位

v0.1.6 聚焦练习模块重构。

本版本的核心目标不是增加更多题，而是让标准练习从“小题线性提交”升级为“大题组验证 + 错题沉淀 + 针对性复练”的闭环。

目标链路：

```text
大题组作答 → 组级统一反馈 → 错题即时订正 → 错题集沉淀 → 到期复练 → 掌握后归档
```

练习模块应继续承接单词、课文和语法模块的学习结果，承担正式输出验证的职责。

---

## 二、当前问题

当前练习页已经支持逐题提交、即时反馈和结果页错题展示，但仍存在以下问题：

1. 教材练习天然按“大题 - 小题”组织，现有页面只按小题线性推进，削弱了大题内部的题型连续性。
2. 每做一道小题就展示正确答案，会影响同一大题后续小题的独立检测价值。
3. 错题只出现在结果页，不是一个可持续复练的学习资产。
4. 用户做错后只能查看详情，缺少“只练错题”的高效路径。
5. 错题没有状态生命周期，系统无法判断错题是否已经真正解决。

---

## 三、版本目标

### 1. 保留大题连续作答

同一道大题下的多个小题应先连续完成，再统一提交核对。

### 2. 保护练习检测价值

提交前不展示正确答案，避免用户根据前一小题答案直接套用后续小题。

### 3. 提供组级反馈

大题提交后，系统按小题展示正确、错误、用户答案、正确答案、错因、相关语法和相关课文入口。

### 4. 建立错题集

所有判错的小题都应进入错题集。错题集不是结果页附属信息，而是后续复练入口。

### 5. 支持错题复练和归档

错题被再次练习并连续正确后，才从重点错题中移出。

---

## 四、练习提交策略

### 结论

练习模块采用“大题组提交”为默认策略。

同一道大题下的小题一次性完成后，再统一核对答案。

### 理由

1. 大题内小题通常共享同一语法点、变形规则、词库或句型。
2. 逐小题即时展示答案会降低后续小题的独立检测价值。
3. 大题组提交能保留教材练习的结构感。
4. 反馈仍限制在一道大题内，不会让用户等到整课结束才知道结果。

### 例外

以下情况可继续采用单题提交：

1. 单独从语法页跳转来的孤立练习。
2. 错题集复练模式。
3. 听力题因素材缺失无法判分时，只记录完成并继续。
4. 未来需要口语录音即时评分的题型。

---

## 五、练习页主流程

### 1. 进入练习页

默认展示当前大题，而不是当前小题。

页面应包含：

1. 课程标题
2. 大题标题
3. 题型分类
4. 作答要求
5. 例题
6. 大题内小题列表
7. 大题完成进度
8. 结果页和错题集入口

### 2. 大题作答

大题内每个小题独立展示：

1. 小题序号
2. 题干
3. 输入框、选择项或题型对应控件
4. 已作答状态

提交前不展示正确答案、参考答案和错因。

### 3. 提交本大题

用户点击“提交本大题”后，系统一次性判定该大题内所有可判分小题。

判题后保存每个小题的结果。

### 4. 组级反馈

提交后展示：

1. 本大题正确数
2. 本大题错题数
3. 每个小题的判定结果
4. 用户答案
5. 正确答案或参考答案
6. 错误说明
7. 相关语法入口
8. 相关课文入口

### 5. 继续路径

组级反馈后提供：

1. 进入下一大题
2. 只重做本大题错题
3. 查看错题集
4. 查看结果页

---

## 六、错题集

### 页面定位

错题集用于承接练习中的薄弱项，让用户后续只练自己做错过、尚未稳定掌握的题。

错题集应同时支持：

1. 本课错题
2. 后续跨课程错题
3. 按语法点筛选
4. 按状态筛选

当前版本优先实现本课错题。

### 错题来源

以下情况进入错题集：

1. 标准练习小题判错。
2. 错题复练再次判错。

以下情况不进入错题集：

1. 听力题因缺少素材被跳过。
2. 造句题没有可判定规则，只提供参考答案时。
3. 用户未提交的题。

---

## 七、错题状态模型

错题至少记录以下字段：

```js
{
  lessonId: "lesson-38",
  exerciseId: "ex-i-1-2",
  groupId: "i-1",
  userAnswer: "用户最近一次错误答案",
  correctAnswer: "标准答案",
  relatedGrammar: ["g1"],
  relatedSentences: ["s1"],
  wrongCount: 1,
  correctStreak: 0,
  firstWrongAt: "ISO 时间",
  lastWrongAt: "ISO 时间",
  lastPracticedAt: "ISO 时间",
  dueAt: "ISO 时间",
  status: "active"
}
```

### 状态说明

```text
active    当前重点错题
reviewing 已订正，但仍需后续复练
resolved  已连续正确，暂时归档
```

### 状态流转

1. 标准练习做错：进入 `active`
2. 错题复练做错：保持 `active`，`wrongCount + 1`
3. 错题复练做对一次：进入 `reviewing`，`correctStreak + 1`
4. 错题复练连续做对两次：进入 `resolved`
5. 已归档错题再次做错：回到 `active`

---

## 八、错题集练习模式

错题集练习采用单题提交。

原因：

1. 错题复练目标是快速确认是否修正。
2. 用户已经见过这道题的反馈，不需要再保护同组后续题。
3. 即时反馈更适合纠错场景。

错题集单题反馈应包含：

1. 是否正确
2. 用户答案
3. 正确答案
4. 错误说明
5. 当前错题状态
6. 连续正确次数

---

## 九、结果页调整

结果页继续保留错题列表，但职责变为本课总结。

结果页必须提供：

1. 完成大题数
2. 完成小题数
3. 正确小题数
4. 当前重点错题数
5. 进入错题集按钮

结果页中的错题列表应优先展示 `active` 和 `reviewing` 状态的错题。

---

## 十、数据与存储

当前原型继续使用本地存储。

建议新增：

```text
lesson:{lessonId}:exerciseGroupAnswers
lesson:{lessonId}:exerciseGroupSubmitted
lesson:{lessonId}:wrongBook
lesson:{lessonId}:wrongPractice
```

原有 `lesson:{lessonId}:exerciseResults` 可继续保留，用于兼容结果页统计。

---

## 十一、实施范围

### 本版本必须实现

1. 练习页按大题组展示。
2. 大题内多个小题一次性提交。
3. 提交后展示组级反馈。
4. 错题自动进入错题集。
5. 结果页提供错题集入口。
6. 错题集支持单题复练。
7. 错题状态支持 `active / reviewing / resolved`。

### 本版本暂不实现

1. 跨课程错题集。
2. 智能间隔复习算法。
3. AI 生成追加练习。
4. 主观造句题深度评分。
5. 教材听力音频缺失时的自动补全。

---

## 十二、验收标准

1. 用户进入练习页后，看到的是一个大题组，而不是单个小题。
2. 同一大题内多个小题可以先全部作答。
3. 提交本大题前不显示正确答案。
4. 提交本大题后，每个小题都有明确反馈。
5. 做错的小题会进入错题集。
6. 错题集可以只练错题。
7. 错题复练做对后会更新连续正确次数和状态。
8. 连续正确两次后，错题从重点错题中归档。
9. 结果页仍能展示本课表现，并能进入错题集。
