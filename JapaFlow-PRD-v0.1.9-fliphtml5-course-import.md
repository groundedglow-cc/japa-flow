# JapaFlow PRD v0.1.9 - FlipHTML5 教材页图导入能力增强

## 1. 背景

当前课程初始化页已经支持用户手动提供图片或 JSON，再由系统确认课程结构并生成音频。问题是图片提供过程仍偏手动：用户需要自己截取、上传、整理教材页，导入一课的准备成本较高。

本版本先验证通过 FlipHTML5 在线教材地址自动获取教材页图的可行性，并将其纳入后续课程初始化能力规划。目标不是一次性导入整本书，而是先支持用户指定课次，例如“导入第 29 课”，系统自动定位对应页图并进入既有初始化流程。

## 2. 能力验证结论

验证地址：

`https://online.fliphtml5.com/cialx/cvaz/#p=70`

验证结果：

- 页面 HTML 可直接请求，标题为“新版中日交流标准日本语初级（下）”。
- 页面源码暴露 `javascript/config.js` 和页图预加载路径 `files/large/1.webp`。
- `config.js` 中可获得基础元信息：`pageCount: 421`、`pageWidth: 470`、`pageHeight: 707`。
- 页图资源路径稳定：`https://online.fliphtml5.com/cialx/cvaz/files/large/{page}.webp`。
- 抽样验证 `files/large/70.webp` 返回 `200 OK`，类型为 `image/webp`。
- 抽样下载并查看 `71.webp`，可以识别为第 29 课起始页，标题为“第 29 课 電気を消せ”。
- 抽样下载并查看 `72.webp`，可以识别为第 29 课练习页开始。
- `files/search/search_config.js` 存在，但 `textForPages` 全为空，不能依赖内置搜索索引定位课文。
- 常见文本层路径如 `files/basic-html/page70.html`、`files/mobile/javascript/text.js` 不存在或返回 404。

结论：自动获取页图可行；直接解析 PDF 文本或页面文本层不可行。后续应采用“页图抓取 + OCR/视觉模型识别 + 用户确认”的方案。

## 3. 目标

v0.1.9 后续实现目标：

1. 用户在课程初始化页输入 FlipHTML5 地址和目标课次。
2. 系统自动读取书籍元信息和页图 URL 模式。
3. 系统自动定位目标课次起止页。
4. 系统将目标课次页图交给现有 AI 解析流程，生成课程初始化 JSON 草稿。
5. 用户确认 JSON 后，再按现有流程生成音频和课程数据。

## 4. 非目标

- 不在本版本实现整本书一键导入。
- 不绕过用户确认直接写入课程数据。
- 不把教材全部长期缓存到项目仓库。
- 不承诺所有 FlipHTML5 书籍都具备相同结构；本版本先面向当前教材地址验证。
- 不改造现有音频生成链路，只复用已有确认后生成音频的流程。

## 5. 用户流程

1. 用户进入课程初始化页。
2. 选择“从在线教材导入”。
3. 输入教材地址，例如 `https://online.fliphtml5.com/cialx/cvaz/#p=70`。
4. 输入目标课次，例如 `29`。
5. 系统展示识别结果：
   - 教材标题
   - 总页数
   - 识别到的目标课次起始页
   - 候选结束页
   - 将用于解析的页图列表
6. 用户确认页范围。
7. 系统调用模型解析页图，生成课程 JSON 草稿。
8. 用户检查并确认课程 JSON。
9. 系统调用音频生成能力，完成课程初始化。

## 6. 课次定位策略

### 6.1 初始策略

由于该来源没有可用文本层，课次定位需要基于图片识别。

建议采用两阶段策略：

1. 粗定位：优先扫描可能包含课次标题的页面。
   - 如果 URL 带 `#p=70`，可从该页附近开始扫描。
   - 如果用户只输入书籍地址，则按页图缩略扫描或分段扫描。
2. 精定位：调用视觉模型识别页面顶部或标题区域，查找类似“第 29 课”的标题。

本次验证中：

- `p=70` 对应印刷页码 54，是第 29 课前一页的语法/词汇内容。
- `p=71` 对应印刷页码 55，是第 29 课起始页。
- `p=72` 对应印刷页码 56，是第 29 课练习页。

### 6.2 结束页判断

优先规则：

1. 找到下一课标题页，例如“第 30 课”，前一页即为第 29 课候选结束页。
2. 如果未找到下一课，使用用户确认页范围。
3. 如果模型信心不足，必须停在确认步骤，不自动进入生成。

## 7. 数据结构草案

在线教材导入任务需要记录以下信息：

```json
{
  "sourceType": "fliphtml5",
  "sourceUrl": "https://online.fliphtml5.com/cialx/cvaz/",
  "bookTitle": "新版中日交流标准日本语初级（下）",
  "pageCount": 421,
  "pageImagePattern": "https://online.fliphtml5.com/cialx/cvaz/files/large/{page}.webp",
  "targetLesson": 29,
  "detectedStartPage": 71,
  "detectedEndPage": null,
  "candidatePages": [71, 72],
  "requiresUserConfirmation": true
}
```

## 8. 解析质量要求

模型生成课程 JSON 前，系统应保留人工确认环节。确认页至少展示：

- 识别到的课次标题。
- 页图缩略预览。
- 起止页范围。
- 将提取的内容类型：课文、词汇、语法、练习。
- 风险提示：页图来源无文本层，识别结果可能存在漏字、错字、假名标注错误。

## 9. 风险

### 9.1 技术风险

- FlipHTML5 不同书籍资源结构可能不同，页图路径不一定总是 `files/large/{page}.webp`。
- 当前教材搜索索引为空，无法用文本搜索直接定位课次。
- OCR/视觉模型可能漏识别小号假名、画线、题号、表格和图片题。
- 页码存在偏移：FlipHTML5 页码、印刷页码、课程页码并不一致。

### 9.2 产品风险

- 自动定位错误会导致导入错课或漏页，因此必须先用户确认。
- 整课页数可能较多，模型解析成本和等待时间需要可视化。
- 图片题、听力题、替换练习的结构化结果仍需要重点校验。

### 9.3 合规风险

- 教材内容可能受版权保护。产品应要求用户确认其拥有合法使用来源和导入权限。
- 系统不应默认公开、分发或长期保存整本教材内容。
- 仅保存课程学习所需的用户确认后结构化结果和必要资源。

## 10. MVP 范围

第一阶段只做第 29 课单课导入验证：

1. 支持输入 FlipHTML5 地址。
2. 自动读取 `config.js`，获取标题、页数和页图路径。
3. 支持按目标课次扫描页图。
4. 自动给出目标课次候选起始页。
5. 用户手动确认起止页。
6. 复用现有图片导入到课程 JSON 的模型解析能力。
7. 复用现有音频生成确认流程。

## 11. 验收标准

- 输入当前 FlipHTML5 地址和课次 `29` 后，系统能识别书籍标题和总页数。
- 系统能构造并访问第 29 课候选页图 URL。
- 系统能将 `71.webp` 识别为第 29 课起始页候选。
- 系统不会在用户确认前写入课程数据或生成音频。
- 用户确认页范围后，后续流程与现有“图片导入课程初始化”一致。

## 12. 推荐实施顺序

1. 新增 FlipHTML5 来源解析器，只负责读取配置和构造页图 URL。
2. 新增课次定位服务，先支持视觉模型识别课标题。
3. 在课程初始化页新增“在线教材导入”入口。
4. 增加页范围确认 UI。
5. 将确认后的页图列表接入现有图片解析流程。
6. 观察第 29 课导入结果，再决定是否扩展到整本书导入。
