# JapaFlow 0.1.7 产品功能说明书（首页课程化重构）

## 一、背景

当前 JapaFlow 的学习体验已经围绕第 27 课形成了较完整的单课闭环，包括单词、语法、课文、练习、结果和音频管理。但首页仍然直接服务于单课原型，用户进入应用后缺少“选择课程”的入口，也无法从全局角度看到多个课次的学习状态。

0.1.7 的目标是把应用从“单课原型”调整为“多课程学习系统”的入口形态。实际学习内容仍先以第 27 课为完整样板，后续课程可以逐步初始化数据。

## 二、目标

1. 首页成为课程列表页，而不是第 27 课的学习入口页。
2. 用户可以从首页看到多个课程卡片，并进入具体课程。
3. 具体课程拥有课程首页，用于展示该课内部学习路径和模块进度。
4. 现有单词、语法、课文、练习、结果、音频管理页面路径保持稳定。
5. 未初始化课程需要明确展示“待初始化”，避免用户误以为可以学习。

## 三、信息架构

### 1. 全局首页 `/`

首页展示课程列表，页面职责是“选课”和“总览”。

课程卡片包含：

- 课号
- 标题或核心句
- 数据状态：可学习 / 待初始化
- 学习状态：未开始 / 学习中 / 已完成 / 需复习
- 模块进度摘要：单词、语法、课文、练习
- 主按钮：继续学习 / 开始学习 / 待初始化

首页不展示具体学习内容，不直接进入单词页。

### 2. 课程首页 `/lesson/:lessonId`

课程首页展示该课的学习仪表盘，页面职责是“进入学习路径”和“判断下一步”。

展示模块：

- 单词：掌握数 / 总数
- 语法：已通过语法点 / 总数
- 课文：已通过朗读句 / 总句数
- 练习：已完成题 / 总题数
- 结果：弱项数量和复盘入口

推荐主路径：

单词 → 语法 → 课文 → 练习 → 结果

但用户可以自由进入任何模块。

### 3. 课程内部模块

以下路径保持不变：

- `/lesson/:lessonId/vocab`
- `/lesson/:lessonId/grammar`
- `/lesson/:lessonId/text`
- `/lesson/:lessonId/exercises`
- `/lesson/:lessonId/result`
- `/lesson/:lessonId/audio`

## 四、数据设计

### 1. 课程目录

新增课程目录元信息，用于首页展示：

```js
const lessonCatalog = [
  {
    id: 27,
    title: "第27课",
    subtitle: "子供の時、大きな地震がありました",
    status: "ready"
  },
  {
    id: 28,
    title: "第28课",
    subtitle: "待初始化",
    status: "pending"
  }
];
```

### 2. 单课数据

当前第 27 课仍使用现有 `lesson` 数据结构。后续多课扩展时，应逐步改为：

```js
const lessons = [lesson27, lesson28];
function lessonById(id) {}
```

0.1.7 不强制完成多课数据迁移，只完成入口和页面结构的课程化。

## 五、交互要求

### 首页

1. 点击可学习课程卡片或按钮，进入 `/lesson/27`。
2. 未初始化课程不可进入学习模块，按钮置灰。
3. 首页课程卡片要能快速判断课程状态。

### 课程首页

1. 展示该课学习总览。
2. 展示模块卡片和进度。
3. 主按钮根据当前进度推荐下一步：
   - 单词未完成：继续单词
   - 语法未完成：进入语法
   - 课文未完成：进入课文
   - 练习未完成：进入练习
   - 全部完成：查看结果
4. 顶部导航在课程内部显示模块入口；在全局首页只保留首页和必要的全局控制。

## 六、验收标准

1. `/` 展示课程列表，而不是单课学习 Hero。
2. `/lesson/27` 展示第 27 课课程仪表盘。
3. 从 `/lesson/27` 可以进入单词、语法、课文、练习、结果、音频。
4. 现有 `/lesson/27/vocab` 等路径不受影响。
5. 未初始化课程有明显的待初始化状态。
6. 代码检查通过，页面可访问。
